## Architecture at a Glance

```
Data Pipeline (Python)
    ↓ Collects MT5 + OANDA prices
    ↓ Fetches economic calendar
    ↓ Validates data quality
    ↓
JSON Context File
    ↓ Machine-readable, source of truth
    ↓ Validator: GO or BLOCKED
    ↓ Setup candidate (if GO)
    ↓
Claude Code Layer
    ↓ Reads JSON (never guesses levels)
    ↓ Cites all claims from JSON
    ↓ Respects GO/BLOCKED decision
    ↓
Report (Markdown)
    ↓ Saved to reports/ with timestamp
    ↓ Trader reads, confirms on chart
    ↓
Manual Trade Log
    ↓ If executed: record entry, exit, P&L
    ↓ Audit trail for performance analysis
```

## Key Principles (Locked In)

1. **Python calculates** → Claude interprets (never inverts)
2. **Protection first** → Risk engine before analysis
3. **JSON is truth** → Claude cites fields, never invents
4. **BLOCKED early** → Prefer missing setup to wrong execution
5. **Fully auditable** → Every trade reconstructible: data → decision → result
6. **Demo first** → 50+ trades shadow + 4-6 weeks demo before FTMO real
